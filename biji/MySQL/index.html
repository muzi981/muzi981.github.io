二、SQL注释

  • 行注释： –注释
  • 多行注释： /* 注释 */

三、SQL常用数据类型

  • 整数：int int unsigned，代表设置一个无符号的整数；
  • 小整数：tinyint tinyint unsigned，代表一个无符号的小整数；
  • 小数：decimal 如decimal(5,2) ,表示共存5个数，小数占2位，不能超过2位；整数占3位，不能超过3位；
  • 字符串：varchar 如varchar(3)：表示最多存储3个字符，一个中午或一个字母都占一个字符；
  • 日期时间： datetime

四、约束

  • 主键（primary key）：能唯一标识表中的每一条记录的属性值 一般在一张数据表中只需要有一个主键就可以了 ；主键的作用：主要用于保证数据表内的数据每一条的顺序是固定的，不会由于删除或者增加数据，而导致数据乱序；
  • 非空（not null）：此字段不允许填写空值
  • 唯一（unique）：此字段的值不允许重复 （如身份证输入就必须唯一）
  • 默认值（default）：当不填写此值时会使用默认值，如果填写时以填写为准
  • 外键（foreign key）：一个表中的一个字段引用另一个表的主键

五、数据库中的元素

  • 数据库：——database

  • 表（table）：

    表（table），数据库中存储数据的基本单位，表是一个由行和列组成的二维

  • 字段（列）：—field

  • 记录（行）：–record

  • 扩展1：逻辑删除（又名：假删）

    对于重要的数据，不能轻易执行delete 语句进行删除。因为一旦删除，数据将无法恢复，所有可以继续逻辑删除。

    1. 给表添加字段，代表数据是否删除，一般起名isdelete，0代表未删除，1代表删除，默认为0；

    2. 当要删除某条数据时，只需要设置这条数据的isdelete字段为1

      • 如果要删除一条数据时，这样做其实对于这样的一条数据在数据库中是没有被删除，只不过通过表示来，提示这个数据已删除；
        • 标记为删除的数据

    3. 以后在查询数据时，只查询isdelete为0的数据

  • 拓展2：其他数据删除方法

    1. delete from 表名 :

      delete from student;

      删除所有数据，但是不重置主键字段的计数；

    2. truncate table 表名;

      truncate table student;

      删除所有数据，并且重置主键字段的计数;

    3. drop table 表名;

      drop table student；

      直接删除整个数据表，意味着字段和数据均不存在，整个表都没了；
